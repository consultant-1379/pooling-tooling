Overview

This page describes the Test Environments section in the Resource Pooling Tool.

It describes what the purpose of this feature is and how it works.

Purpose

The purpose of having a test environments section in RPT is so that there can be a clear view of all test environments and their state.

It provides the user of RPT a UI which clearly displays information about each environment.

Test Environments Table

The test environments table is where the test environments and the information related to each environment is displayed.

The table is updated with the test environment live status from pipelines.

  • Available:

    The test environment is free and ready to use.

  • Reserved:

    The test environment is currently being used, typically its running through a pipeline.

  • Quarantine:

    The test environment is in a quarantine state. For a test environment to be in Quarantine state means something is wrong with the test environment or something is failing. Typically, a test environment will be placed in a Quarantine state as a result of a failed pipeline.

  • Standby:

    A test environment in Standby will not be picked up to be used by CI/CD pipelines. It will however be refreshed daily by a refresh pipeline triggered by RPT to keep it up to date. If the sticky environment gets quarantined, the freshest Standby environment will become available.

  • Refreshing:

    A test environment in the "Refreshing" state is currently being refreshed and should not be interrupted.

Each test environment can be part of a pool. The name of the assigned pool is also displayed in the table


Heading Description
Name The name of a test environment.
Status The status a test environment is in: Available, Reserved, Quarantine, Standby or Refreshing.
Pools The pools that that test environment is a part of.
Properties This is a link that will open a modal with the information about the test environment properties.
Pipeline Stage If a test environment is running through a pipeline, the current stage the test environment is on is displayed here.
Additional Info This provides the live status from pipelines and ownership of the test environment. When provided it captures information about the current status and the agent responsible for initiating the change of status.
Comments This is a textarea which will allow a user to input other information they may want to store about a test environment that will persist until overwritten.

Change Status

You may want to move a test environment to Available status to show it is ready to be used.

You may also realize there is a problem and you want to manually set it into Quarantine.

Lastly, you may want to use the test environment in a pipeline and therefore you must change the status to Reserved.

To do these actions:

  1. Click the three buttons under Actions in the test environment table
  2. Hover over "Environment Actions"
  3. Click the appropriate "Change Status" button

Re-order Elements

You may want to re-order the test environments as they appear in the table.

A table row is draggable if the move cursor appears as you hover over the environment in the table.

To re-order the elements, simply drag and drop the selected test environment to the desired position in the table.

Please note that the ordering of the elements in the table has no impact on behavior of the system.

Copy Environment Names

You may want to copy the name of a test environment to your clipboard.

This can be done in two ways:

  • Right clicking the name of the test environment and selecting 'Copy'.

  • Opening the environment properties view and double-clicking the name of the test environment before selecting 'Copy'.

Copy Additional Info

You may want to copy the contents of 'Additional Info' to your clipboard.

This can be done by right clicking the the contents of the 'Additional Info' cell and selecting 'Copy'.

Copy Comments

You may want to copy the contents of 'Comments' to your clipboard.

Please note that this can only be done while the 'Comments' is in edit mode.

You can enter edit mode by clicking the pen located to the right, and exit edit mode by clicking the tick.

Comments Icons

  • Edit comments (edit) :

    By clicking this, you enter edit mode and can now edit the content of 'Comments'.

  • Undo changes to comments (undo) :

    When clicking this icon, the content of 'Comments' will revert the changes made to it while in edit mode.

  • Confirm changes to comments (done) :

    Click to confirm any changes made to 'Comments' while in edit mode.

  • Expand comments (arrow_downward) :

    This is used to expand the 'Comments' section in the case that the content exceeds the default line length.

  • Collapse comments (arrow_upwards) :

    After expanding the 'Comments', you can then collapse the content by clicking this icon.
