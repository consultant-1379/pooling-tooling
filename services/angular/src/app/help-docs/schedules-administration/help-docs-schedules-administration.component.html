Overview

This page describes the schedule administration features in RPT.

It describes the purpose of these features and how they work.

Purpose

The purpose of having schedules administration section in RPT is to provide the ability to:

  • Add Schedule
  • Edit Schedule
  • Remove Schedule

Add Schedule

Steps

1 Enter schedule name in the search box

  • Please note that this is case sensitive.

2 Click on the 'Add Schedule' button

  • If the schedule exists dialog box will be populated stating "Schedule * already exists in RPT".
  • If the schedule does not exist, Add Schedule form will be populated.

3 Please enter the schedule's properties

    Heading Description
    Schedule Enabled Specifies if the schedule is enabled or disabled, the default position of the switch is set to off
    Type Of Items To Schedule The type of item you want to schedule (e.g. Pool, Test-environment)
    Changing this setting will clear any items selected in the "Items To Schedule Ids" dropdown.
    Items To Schedule Ids The pools or test environments the schedule will be associated with. This is a dropdown menu and multiple items can be selected.
    Spinnaker Pipeline Application Name The Spinnaker application the schedule will be associated with.
    Spinnaker Pipeline Name The Spinnaker pipeline the schedule is associated with.
    Retention Policy Enabled Enable a retention policy for the "auto-refresh" schedule, the default position of the switch is set to off.

    The retention policy ensures that a specified number of standby environments is kept in RPT as specified in "No. Of Standby Envs to be Retained".
    After the number of standby environments are retained the rest of the environments are then compared to the specified number of recently released software versions, if the environment version is older it is then refreshed.
    If the retention policy is enabled the following properties will need to be populated.

    • No. Of Standby Envs to be Retained
    • No. Of Software Release Versions For Comparison
    No. Of Standby Envs to be Retained States the number of standby environments will be excluded from being refreshed
    No. Of Software Release Versions For Comparison States the number of release versions to compare against and refreshes any environments on older versions.
    Schedule Type The type of schedule. This is a dropdown menu where two schedule types can be selected.
    • auto-refresh:

      This shcedule is used when kicking off a refresh spinnaker pipeline. This schedule chooses stanby environments to use.

    • auto-trigger:

      This schedule is used when triggering any other spinnaker pipeline. This schedule chooses available environments to use.
    Project Area The project area the schedule will be associated with (e.g. pso ,prod-eng). This is a dropdown menu and a single item can be selected.
    If your selection is not available please contact team Thunderbee.
    Cron Schedule The cron schedule is a set of five fields in a line, indicating when the job should be executed.
    The input syntax is (minute, hour, day of the month, month, day of the week) with a space between each input. Inputs are checked with a message.

4 Click on 'Add Schedule'

Edit Schedule

Steps

1 Enter schedule name in the search box

  • Please note that this is case sensitive.

2 Click the 'Edit Schedule' button

  • If the schedule exists, a dialog box will be populated with the schedule's current attributes.
  • If the schedule does not exist, the user will be notified by a pop-up window.

3 Edit the schedule's properties

    Heading Description
    Schedule Enabled Edit if the schedule is enabled or disabled.
    Type Of Items To Schedule Edit the type of item you want to schedule (e.g. Pool, Test-environment)
    Changing this setting will clear any items selected in the "Items To Schedule Ids" dropdown.
    Items To Schedule Ids Edit the pools or test environments the schedule will be associated with.
    Spinnaker Pipeline Application Name Edit the Spinnaker application the schedule will be associated with.
    Spinnaker Pipeline Name Edit the Spinnaker pipeline the schedule is associated with.
    Retention Policy Enabled Edit if the retention policy on the schedule is enabled or disabled.

    The retention policy ensures that we keep a specified number of standby environments in RPT as specified in "No. Of Standby Envs to be Retained".
    After the number of standby environments are retained the rest of the environments are then compared to the specified number of recently released software versions, if the environment version is older it is then refreshed.
    If the retention policy is enabled the following properties will need to be populated.

    • No. Of Standby Envs to be Retained
    • No. Of Software Release Versions For Comparison
    No. Of Standby Envs to be Retained States the number of standby environments will be excluded from being refreshed
    No. Of Software Release Versions For Comparison States the number of release versions to compare against and refreshes any environments on older versions.

    We refresh if the version on the environment is not within the last specified number of recently released software versions.
    Schedule Type Edit the type of schedule (e.g. auto-refresh)
    Project Area Edit the project area the schedule will be associated with (e.g. pso ,prod-eng)
    If your selection is not available please contact team Thunderbee.
    Cron Schedule Edit the cron schedule, this changes when the job should be executed.
    The input syntax is (minute, hour, day of the month, month, day of the week) with a space between each input. Inputs are checked with a message.

4 Click on 'Edit Schedule' to modify the schedule

Remove Schedule

Steps

1 Enter schedule name in the search box

  • Please note that this is case sensitive.

2 Click on the 'Remove Schedule' button

  • If the schedule does not exist, a dialog box will pop up stating that the schedule does not exist in RPT.
  • If the schedule exists, the Remove Schedule form will be opened.

3 Re-enter the schedule's name

  • To confirm, please re-enter the name of the schedule to be removed.
  • The submit button will only be enabled in the name of the schedule to be removed has been re-entered.

4 Click on the 'Submit' button to remove the Schedule
