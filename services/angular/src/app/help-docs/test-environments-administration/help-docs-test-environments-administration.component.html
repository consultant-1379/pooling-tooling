Overview

This page describes the test environments administration features in RPT.

It describes the purpose of these features and how they work.

Purpose

The purpose of having test environments administration section in RPT is to provide the ability to:

  • Add Test Environments
  • Edit Test Environments
  • Remove Test Environments

Add Test Environment

Steps

1 Enter test environment name in the search box

  • Please note that this is case sensitive.

2 Click on the 'Add Environment' button

  • If the environment exists dialog box will be populated stating "Test Environment env already exists in RPT".
  • If the environment does not exist, Add Environment form will be populated.

3 Please enter the test environment's properties

  • Core Information Description
    Pool Select appropriate pools to which the test environment belongs.
  • Cluster Properties Description
    Product The area your test environment is associated with (e.g. EIC).
    Platform Type The platform type of test environment (e.g. Microsoft Azure, CNIS).
    Version The version of the application that is installed on the test environment.
    CCD Version The version of CCD that is installed on the test environment.

4 Click on 'Add Environment'

Edit Test Environment

Steps

1 Enter test environment name in the search box

  • Please note that this is case sensitive.

2 Click the 'Edit Environment' button

  • If the environment exists, a dialog box will be populated with the environment's current attributes.
  • If the environment does not exist, the user will be notified by a pop-up window.

3 Edit the test environment's properties

  • Core Information Description
    Pool Select appropriate pools to which the test environment belongs.
  • Cluster Properties Description
    Product Edit the area your test environment is associated with (e.g. EIC).
    Platform Type Edit the platform type of test environment (e.g. Azure).
    Version Edit the version of the application installed on the test environment.
    CCD Version Edit the CCD version installed on the test environment.

4 Click on 'Edit Environment' to modify the environment

Remove Test Environment

Steps

1 Enter test environment name in the search box

  • Please note that this is case sensitive.

2 Click on the 'Remove Environment' button

  • If the environment does not exist, a dialog box will pop up stating that the environment does not exist in RPT.
  • If the environment exists, the Remove Environment form will be opened.

3 Re-enter the environment's name

  • To confirm, please re-enter the name of the environment to be removed.
  • The submit button will only be enabled in the name of the environment to be removed has been re-entered.

4 Click on the 'Submit' button to remove the Environment
