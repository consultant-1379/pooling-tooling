Overview

This page describes the Schedules section in the Resource Pooling Tool.

It describes what the purpose of this feature is and how it works.

Purpose

The purpose of having a schedules section in RPT is so that there can be a clear view of all schedules and their information.

It provides the user of RPT a UI which clearly displays information about each schedule.

Schedules Table

The schedules table is where the schedules and the information related to each schedule is displayed.


Heading Description
Name The name of a test environment.
Project Area The project area a schedule is in.
Environments The pools or test environments the schedule is associated with.
Cron Schedule The cron schedule displays the frequency at which the schedule is executed.
It is output in a readable form with the five fields of the command displayed in brackets (minute, hour, day of the month, month, day of the week)
Spinnaker Application The Spinnaker application the schedule is associated with.
The text is linked to the spinnaker application page.
Spinnaker Pipeline The Spinnaker pipeline that the schedule is associated with.
The text is linked to the spinnaker pipeline page.
Status Specifies if the schedule is enabled or disabled
Type Of Items Specifies the type of item the schedule checks.
Retention Policy Status Displays if the schedule has a retention policy associated with it.

The retention policy ensures that we keep a specified number of standby environments in RPT as specified in "No. Of Standby Envs to be Retained".
After the number of standby environments are retained the rest of the environments are then compared to the specified number of recently released software versions, if the environment version is older it is then refreshed.
If the retention policy is enabled the following properties will need to be populated.

  • No. Of Standby Envs to be Retained
  • No. Of Software Release Versions For Comparison
No. Of Standby Envs to be Retained States the number of standby environments will be excluded from being refreshed
No. Of Software Release Versions For Comparison States the number of release versions to compare against and refreshes any environments on older versions.
Schedule Type Displays the schedule type
Full Info This is a link that will open a modal with all properties of the schedule.

The following properties are viewable in this modal.

  • Type Of Items
  • Retention Policy Status
  • No. Of Standby Envs to be Retained
  • No. Of Software Release Versions For Comparison
  • Schedule Type
