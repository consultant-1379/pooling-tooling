Overview

This page describes the Pools section in the Resource Pooling Tool.

It describes what the purpose of this feature is and how it works.

Purpose

The purpose of having a Pools section in RPT is to provide a view of all test environments in a particular pool and their state.

It provides the user of RPT a UI which clearly displays information about each environment associated with a pool.

Pools

The Pools table displays a subset of test environments belonging to a selected pool along with the information related to the environment.

To view the test environments in a pool select a 'pool' from the dropdown 'Pools' menu in the navigation bar. The page can be bookmarked for the desired pool.

The table is updated with the test environment live status from pipelines.

For more information about on the test environment table check out 'Test Environments' section under 'Help Docs'.

Re-order Elements

You may want to re-order the test environments as they appear in the table.

A table row is draggable if the move cursor appears as you hover over the environment in the table.

To re-order the elements, simply drag and drop the selected test environment to the desired position in the table.

Please note that the ordering of the elements in the table has no impact on behavior of the system.

Copy Environment Names

You may want to copy the name of a test environment to your clipboard.

This can be done in two ways:

  • Right clicking the name of the test environment and selecting 'Copy'.

  • Opening the environment properties view and double-clicking the name of the test environment before selecting 'Copy'.

Copy Additional Info

You may want to copy the contents of 'Additional Info' to your clipboard.

Please note that this can only be done while the 'Additional Info' is in edit mode.

You can enter edit mode by clicking the pen located to the right, and exit edit mode by clicking the tick.
